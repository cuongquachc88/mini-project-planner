User Guide

Everything you need to get the most out of Mini Project Planner — from first install to advanced vault usage.

Installation

Mini Project Planner is a Progressive Web App (PWA). No app store needed — install directly from your browser.

🖥️

Desktop (Chrome / Edge)

Click the install icon in the address bar, or open the browser menu and choose Install Mini Project Planner.

📱

iOS (Safari)

Tap the Share button → Add to Home Screen. The app opens in standalone mode without the browser chrome.

🤖

Android (Chrome)

A banner appears automatically. Tap Install, or use the browser menu → Add to Home screen.

Offline support: Once installed, the app works 100% without an internet connection. All data is stored in your browser's IndexedDB.

Your First Project

  1. 1
    Set up your profile Click the avatar in the top-right → Profile. Enter your display name and email. You'll be the project admin.
  2. 2
    Create a project From the Home screen, click New Project. Give it a name, choose a key (e.g. SHOP), pick an icon and color. The key prefixes all work item IDs (e.g. SHOP-1).
  3. 3
    Open the board You'll land on the Kanban board. Default stages: To Do → In Progress → Done. You can rename, reorder, or add stages in Project Settings.
  4. 4
    Add team members Go to Settings → Team Members to add collaborators. Members can be assigned to work items.

PIN Lock

Protect the app with a 6-digit PIN. When set, the app shows a lock screen on every startup.

  1. Go to Profile → PIN Lock and click Set PIN.
  2. Enter a 6-digit PIN and confirm it. The PIN is hashed with SHA-256 and never leaves your device.
  3. To change or remove the PIN, you must enter the current PIN first.
Important: There is no PIN recovery. If you forget your PIN, you'll need to clear the app's browser data to regain access (this will erase all local data unless you have a Drive backup).

Kanban Board

The board shows all work items in the active sprint, organized into columns by stage.

Drag & Drop

Drag any card between columns to change its stage. The change is saved instantly to the local database.

Filtering

Use the filter bar above the board to narrow cards by:

  • Assignee — show only items assigned to a specific person
  • Type — Story, Bug, Spike, Task, Action, Request
  • Priority — Critical, High, Medium, Low
  • Epic — filter by parent epic

Work Items

Click any card to open the detail panel. Fields you can set:

TypeStory, Bug, Spike, Task, Action, or Request

Sprints

Sprints organize work into time-boxed iterations. The sprint lifecycle has three states:

📋

Planning

Sprint is being set up. Add items from the backlog. Set a goal, start date, and end date.

▶️

Active

Sprint is running. Items appear on the board. The team works through the columns.

✅

Completed

Sprint is closed. Unfinished items can be moved to the backlog or the next sprint. Velocity is recorded.

Starting a Sprint

  1. From the Backlog, select items and assign them to the sprint.
  2. Click Start Sprint in the sprint panel. Set the end date.
  3. The board now shows only items from the active sprint.

Backlog

The Backlog page lists all work items not yet assigned to a sprint. Use it to:

  • Create new items for future sprints
  • Prioritize by dragging items up or down
  • Assign items to the current sprint planning
  • Filter and search across all unstarted work

Epics

Epics are large bodies of work that span multiple sprints. Each epic tracks its own progress.

Creating an Epic

  1. Go to the Epics tab in the sidebar.
  2. Click New Epic. Give it a title, description, and choose a color.
  3. Link work items to the epic from the work item detail panel.

Milestones

Each epic can have milestones. A milestone has a target date and a health indicator:

  • Green — on track
  • Amber — at risk
  • Red — behind / blocked

Project Vault

The Vault is a knowledge base attached to each project. It has 6 sections for different types of project documentation.

Wiki

A tree-structured set of markdown pages. Use it for technical docs, onboarding guides, runbooks, or any reference material.

  • Pages can be nested under parent pages
  • Full rich text editing: headings, lists, code blocks, links
  • Click any page in the tree to view or edit it

Meeting Notes

Log meeting notes with date, attendees, and a rich text body.

  • Notes are sorted by date — most recent first
  • Use the attendees field to record who was present
  • Body supports checklists for tracking action items

Decision Log

Record architectural decisions, product choices, or any significant call the team makes.

Retrospectives

Structure sprint retrospectives with three columns:

  • Went well — what the team should keep doing
  • Improve — what slowed the team down or caused friction
  • Actions — specific action items for next sprint

Each retro is linked to a sprint date so you can look back over time.

Run Sheets

Ordered checklists for repeatable processes — deployments, release procedures, incident response, etc.

  • Add steps in order — each step has a title and optional notes
  • During a run, tick off steps as they're completed
  • Reset the checklist to run it again for the next cycle

Google Drive Sync

Back up and restore your data using Google Drive's private AppData folder. The backup is a single SQL file visible only to this app.

Setup

  1. 1
    App must be configured with a Google Client ID The self-hosted version requires a VITE_GOOGLE_CLIENT_ID environment variable. The hosted version has this preconfigured.
  2. 2
    Connect from Profile Go to Profile → Google Drive Sync and click Connect Google Drive. A popup opens for Google sign-in.
  3. 3
    Authorize the app Grant access to the Drive AppData scope. The app never reads your personal Drive files — only its own backup folder.

Sync behaviour

  • Click Sync now to push a backup immediately
  • The app auto-syncs every 15 minutes when online and connected
  • Sync status (last synced time, errors) shows in the top bar
Multi-device restore: On a new device, connect Google Drive and use the Restore option to pull down the latest backup. The page will reload with your data.

Project Settings

General

  • Project name — display name shown in the sidebar
  • Project key — short prefix for issue IDs (e.g. SHOP). Must be unique.
  • Icon & color — customize how the project appears in the switcher

Board Stages

Add, rename, reorder, or remove columns on the Kanban board. Every project has its own stage configuration.

Team Members

Add team members by name. Members appear in the assignee dropdown when creating or editing work items.

Roles: The first user is set as Admin. Team members added via Settings are assignable but don't have separate login credentials — the app is single-user by design in v1.0.
